# Contributing to UDP Server Manager

Thank you for your interest in contributing!

## How to Contribute
- Fork the repository and create a feature branch.
- Follow the per-device modular structure (see `core/workers/`).
- Write clear, maintainable code and document new features.
- Add or update tests if applicable.
- Submit a pull request with a clear description of your changes.

## Coding Standards
- Use Python 3.8+ and follow PEP8 style guidelines.
- Keep device logic isolated in its own subfolder.
- Use robust error handling and logging.

## Documentation
- Update `README.md` and relevant docs for any user-facing or architectural changes.

## Reporting Issues
- Use the issue tracker to report bugs or request features.
- Provide as much detail as possible for reproducibility.
